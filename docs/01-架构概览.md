# 01 - 架构概览

## 核心理念

Confkit Dashboard 旨在为 `confkit/engine` 命令行工具提供一个图形化用户界面 (GUI)。由于 `engine` 本身是一个 CLI 应用，我们引入一个后端服务作为 **桥梁 (Bridge)**，连接前端 UI 与 CLI。

## 系统组件

系统由三个核心部分组成：

1.  **前端应用 (Frontend App)**
    -   **职责**: 提供用户交互界面，管理视图状态，并通过 HTTP/WebSocket 与后端服务通信。
    -   **技术**: `fukict` (SPA), `Tailwind CSS`。

2.  **后端服务 (Backend Service)**
    -   **职责**: 接收前端的 API 请求，调用 `confkit/engine` 的命令行，解析其输出，并将结构化的数据返回给前端。
    -   **技术**: `Rust`, `axum` (Web框架), `tokio` (异步运行时)。

3.  **Confkit Engine (CLI Tool)**
    -   **职责**: 实际的业务逻辑执行者，负责构建、部署、日志记录等。
    -   **技术**: `Rust` (CLI)。

## 数据流向

```
  用户操作       HTTP/WS 请求        执行 CLI 命令          文件 I/O, Docker 操作
+----------+      +------------+      +----------------+      +--------------------+
|          |----->|            |----->|                |----->|                    |
| Frontend |      | Backend    |      | confkit/engine |      |   系统资源         |
|  (fukict)  |<-----|  (axum)    |<-----|     (CLI)      |      | (YAML, Logs, etc.) |
|          |      |            |      |                |<-----|                    |
+----------+      +------------+      +----------------+      +--------------------+
  UI 更新        JSON 响应           CLI 输出 (stdout)
```
