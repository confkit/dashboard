# 文档策略

## CLAUDE.md 维护

项目结构性变更时, **必须同步更新 CLAUDE.md**:

- 新增/删除/移动重要目录或模块
- 变更项目架构或技术栈
- 修改构建、部署流程
- 新增/变更核心依赖

## 最小化文档

- **不要创建**不必要的文档文件
- **不要自动生成**未经请求的文档
- **先询问**如果认为文档有价值
- **遵循项目结构**如果文档已存在

无需许可即可添加:

- 复杂逻辑的代码注释 (保持精简)
- 公共 API 的 JSDoc/docstring (如果项目已在使用)

## 禁止文档

未经明确请求, 禁止创建:

- 临时文件: `temp.md`、`draft.md`、`notes.md`
- 变更日志: `refactor.md`、`migration.md`、`changelog.md`
- 既定结构外的随意文档
